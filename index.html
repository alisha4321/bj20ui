01 — What is Full Stack Development?
01 / Topic Overview

WHAT IS FULL STACK DEVELOPMENT?

Full Stack Development means building all three layers of a web application — the front-end users see, the back-end server that runs logic, and the database that stores data. A full-stack developer can work across every layer.

① Front-End (Client Side) — HTML, CSS and JavaScript running in the browser. This is everything the user sees and interacts with. Frameworks like Bootstrap and React speed up development.

② Back-End (Server Side) — Languages like Node.js, Python and PHP run on a remote server. They process requests, apply business logic, handle authentication, and talk to the database. The front-end never directly touches data — it asks the back-end via an API.

③ Database (Data Layer) — Where all persistent data lives. SQL databases (MySQL, PostgreSQL) store data in tables. NoSQL (MongoDB) stores flexible JSON documents. All databases support CRUD — Create, Read, Update, Delete.

Full Stack — all 3 layers communicating 
// ① FRONTEND — browser sends a request
const res  = await fetch('/api/products');
const data = await res.json();
renderProducts(data);

// ② BACKEND — Node.js server receives & responds
app.get('/api/products', async (req, res) => {
  const rows = await db.query('SELECT * FROM products');
  res.json(rows);
});

-- ③ DATABASE — SQL returns matching rows
-- SELECT id, name, price FROM products WHERE active = 1;

Reading the code: The browser calls fetch() — that is the front-end sending an HTTP request. The Node.js route receives it, runs a SQL query against the database, and sends results back as JSON. The browser renders that JSON on screen. This cycle is the foundation of every full-stack app.

Responsive Media Queries 
/* Mobile first — single column by default */
.layout { grid-template-columns: 1fr; }

/* Tablet — two columns from 768px */
@media (min-width: 768px) {
  .layout { grid-template-columns: 1fr 1fr; }
}

/* Desktop — three columns from 1024px */
@media (min-width: 1024px) {
  .layout { grid-template-columns: repeat(3, 1fr); }
}

How media queries work: Rules inside @media (min-width: 768px) only apply when the screen is at least that wide. Mobile-first means writing the simplest layout first, then adding more complex layouts as screen size grows. Try resizing your browser to see this page respond.

04 — Bootstrap Framework
04 / Framework

BOOTSTRAP — RAPID COMPONENT UI

Bootstrap is the world's most popular CSS framework. It provides a 12-column responsive grid, ready-made components (modals, accordions, cards) and utility classes. You apply class names in HTML — Bootstrap handles the styling and behaviour. Bootstrap 5 requires no jQuery.

How to add Bootstrap — CDN 
<!-- CSS in <head> -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/css/bootstrap.min.css">

<!-- JS before </body> — powers modals, dropdowns, accordions -->
<script src="https://cdn.jsdelivr.net/npm/bootstrap@5.3.3/dist/js/bootstrap.bundle.min.js"></script>

JAVASCRIPT — BEHAVIOUR & INTERACTIVITY

JavaScript is the programming language of the web. It runs in the browser, manipulates the DOM, responds to events, processes data, and communicates with APIs. Every interactive element on this page is powered by JavaScript.

Core JavaScript Concepts 
// const = never reassigned   let = can change
const name  = "Alisha";
let   score = 0;

// Functions — reusable blocks of logic
function greet(user) {
  return `Hello, ${user}!`;   // template literal
}

// DOM — find an HTML element and listen for a click
document.getElementById('btn')
  .addEventListener('click', () => {
    document.getElementById('out').textContent = greet(name);
  });

// Arrays & loops
['HTML', 'CSS', 'JS'].forEach(s => console.log(s));

How JS connects to HTML: getElementById() finds an element. addEventListener('click', fn) runs a function on click. textContent changes visible text. These three form the foundation of all DOM manipulation.
